# Q-RoPE Transfer Family Survivability Criteria v1

Date: 2026-03-12
Stories: S959

## Purpose
State the current internal criteria for when a new transfer family is likely to survive deeper structural hardening under the same frozen fairness discipline.

## Standing Contrast Set
### Preserved Positive Transfer Families
- `path`
- `loop-closure`
- `fork-join`

### Archived Failure Boundary
- `braid`

## Narrow Internal Rule
A transfer family is more likely to survive deeper reindexing when its useful signal depends on latent-conditioned relational responses that remain non-compressible after nuisance and structural perturbation. A transfer family is more likely to fail when deeper reindexing collapses its effective relational state into a compact summary that a bounded symbolic family can recover.

## Candidate Survivability Criteria
### 1. Deep-Reindex Non-Compressibility
- Under deeper pairing perturbation, the effective relational state should not collapse into a small symbolic surrogate.
- This is the strongest current distinction between `braid` and the preserved families.

### 2. Multi-View Stability
- The target should remain informative under nuisance perturbations such as token renaming and slot exchange.
- This is necessary but not sufficient.
- `braid` survived shallow nuisance hardening and still failed later.

### 3. Structural Rather Than Cosmetic Diversity
- A new transfer family should alter the relational dependency pattern itself, not just restate the same pattern with a new label surface.
- `path`, `loop-closure`, and `fork-join` differ in structural dependency.
- Many archived lines earlier in the repo failed because they only relabeled similar dependency surfaces.

### 4. Frozen Symbolic Recoverability Bound
- A family should be screened at memo level for whether the declared summaries could plausibly compress to the current bounded symbolic ladder.
- If the likely answer is yes, implementation should be blocked.

## Negative Criterion
Do not open a new transfer family if its defining relational state can be written as a shallow recombination of:
- local order summaries
- shallow pairwise agreements
- or braid-like compact crossing summaries
once deeper pairing perturbation is applied.

## Practical Selection Rule
A future transfer family is execution-worthy only if it can satisfy all of the following at memo level:
1. materially different relational dependency pattern from `path`, `loop-closure`, `fork-join`, and `braid`
2. plausible deep-reindex non-compressibility
3. frozen bounded symbolic fairness family declared in advance
4. nuisance and structural hardening plan declared in advance

## Current Best Use
- Use this note as a filter before opening any new transfer family.
- Treat `braid` as the counterexample showing why shallow novelty is not enough.
- Prefer theory-first rejection of weak transfer ideas over more execution churn.
