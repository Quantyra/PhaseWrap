# Q-RoPE Future Transfer Selection Rule v2

Date: 2026-03-12
Stories: S985

## BLUF
A future transfer family should only be approved for execution if it is structurally closer to the preserved survivor cluster than to `braid`.

## Required Positive Conditions
- delayed relational dependence is essential, not decorative
- useful signal depends on accumulated or relayed latent state
- recombination or closure cannot be replaced by shallow local summaries
- memo-level analysis gives a plausible reason deeper reindexing will not collapse the target into a compact symbolic recovery path

## Immediate Reject Conditions
Reject before implementation if the candidate appears reducible to:
- shallow ordering summaries
- compact tuple lookup over declared summaries
- bounded additive/quadratic recovery after deeper perturbation
- a structural pattern already represented by `braid`

## Required Negative Comparison
Before code reopens, every new transfer-family memo must include one explicit paragraph answering:
- why this family is not another `braid`
- what latent-conditioned state is expected to survive deeper reindexing
- which declared summaries are still insufficient under the frozen fairness contract

## Execution Discipline
- one transfer family at a time
- one bounded symbolic control family at a time
- one fixed hardening cycle only
- stop immediately if the family collapses at a braid-like deeper perturbation boundary

## Current Decision
- no fifth transfer execution line should open by default
- any future transfer line must clear this structural screen first
