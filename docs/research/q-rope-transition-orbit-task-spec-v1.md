# Transition Orbit Task Specification v1

## Task
- `synthetic_chart_transition_orbit_response`

## Goal
Define a chart-transition regression task where token identity is quotient-ed into orbit classes before rendering, so token names are nuisance by construction rather than a latent paired-render diagnostic afterthought.

## Construction
- Build latent dual samples from the chart-transition family.
- Replace raw token identities with orbit classes under the fixed renaming group generated by `cdab`.
- Render only orbit-canonical representatives for each latent sample.
- Target depends on:
  - ordered chart-transition structure
  - orbit-transition compatibility
  - sector-magnitude / orientation geometry
- Target must not depend on raw token IDs.

## Required diagnostics
- `orbit_target_invariance_pass = true`
- `orbit_target_max_abs_delta = 0.0`
- `orbit_canonical_balance_pass = true`

- `V_control_symbolic_transition_additive_regressor`
- `V_control_symbolic_transition_unordered_regressor`
- `V_control_symbolic_transition_cross_direction_regressor`
- `V_control_symbolic_transition_quadratic_regressor`
- one future orbit-additive symbolic control
