# Q-RoPE Constrained Publication Framing Memo v1

## Recommended abstract-level framing
Q-RoPE is a relative-phase positional encoding proposal for hybrid quantum attention. The current program supports a coherent novelty claim, a reproducible baseline taxonomy, and real remote execution evidence on both a photonic provider path and an IBM Runtime comparator path. The present remote packet shows feasibility together with strong seed- and provider-sensitivity, so the correct framing is bounded empirical evidence rather than stable remote superiority.

## Safe external statements
1. We propose a relative-phase positional encoding method for hybrid quantum attention.
2. We evaluate Q-RoPE against no-PE and fixed-gate PE baselines in a controlled protocol.
3. We have real remote execution evidence on both a photonic provider path and an IBM Runtime comparator path.
4. Our current remote evidence shows that behavior is strongly seed- and provider-sensitive.
5. The current evidence supports feasibility and boundary-setting, not universal gain claims.

## Unsafe external statements
1. Q-RoPE wins on hardware.
2. Q-RoPE is consistently better than other positional methods.
3. Photonic and superconducting-style providers agree on the ranking.
4. The remote evidence validates robust NISQ superiority.
5. The current packet generalizes beyond the present dataset and slice policy.

## Recommended one-paragraph pitch
Q-RoPE is a Quantyra research program on relative-phase positional encoding for hybrid quantum attention. The method is novel in how it targets a RoPE-like relative-position inductive bias inside a quantum-attention setting, and the repo now contains a reproducible comparator stack spanning no positional encoding, fixed-gate positional encoding, and Q-RoPE. We have executed the program remotely on both a photonic provider path and an IBM Runtime comparator path. Those remote results are valuable because they establish feasibility and expose strong seed/provider sensitivity, which in turn gives us a disciplined, publication-safe claim boundary rather than an overfit “hardware win” story.

## Bottom line
- Publish as a bounded method-plus-evidence paper, not a triumph narrative.
- Lead with novelty, reproducibility, and disciplined empirical boundaries.
- Treat the remote packet as a rigor asset, not a headline performance claim.
