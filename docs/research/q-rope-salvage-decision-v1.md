# Q-RoPE Salvage Decision v1

## Decision
Do not archive the repo as dead.
Preserve it with one explicit restart hypothesis:

- `synthetic theorem-to-mechanism validation`

## Why this is the correct call
The initiative is not strong enough for active productization or publication under the current evidence.
But it is also not empty:
- the core relative-phase idea remains coherent
- the repo infrastructure is strong
- the failed branches tell us where not to spend time

That makes the right state:
- dormant but restartable

Not:
- abandoned

## Chosen restart hypothesis
Restart only if Quantyra chooses to test:
- whether the relative-phase mechanism actually expresses the intended relative-offset inductive bias on tightly controlled synthetic tasks

This is better than immediately restarting with:
- another benchmark expansion
- another local tweak
- another remote wave

## Restart order
1. synthetic theorem-to-mechanism validation
2. if positive, then full query-key comparison redesign
3. if still positive, then better benchmark/task regime

## Immediate program posture
Current posture:
- internal archive
- maintain repo integrity
- no publication push
- no active broad experimentation

Preserved future path:
- one restartable technical hypothesis, clearly named and ordered

## Bottom line
The project should be preserved, not buried.
But it should only be restarted through the synthetic theorem-to-mechanism path.
