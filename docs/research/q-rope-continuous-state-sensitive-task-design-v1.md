# State-sensitive continuous task design

## Motivation
The current continuous task is exhausted because a bounded symbolic lookup over the three agreement bits can solve it more effectively than the witness branch.

## Preserved next angle
- Future task family: `state-sensitive continuous relational response`
- Required property: the target must vary within the same coarse boolean agreement state.
- Disallowed shortcut: any task whose target is fully recoverable from a small one-hot lookup over the existing boolean agreements.

## Design requirement
A future continuous restart should include at least one within-state analog degree of freedom, for example:
- sector-magnitude asymmetry contribution
- ordered token-pair composition contribution
- orientation intensity or graded coupling contribution

## Gate for future approval
A future restart brief must prove, on paper, that a bounded symbolic lookup over coarse agreement states is insufficient before implementation is reopened.
