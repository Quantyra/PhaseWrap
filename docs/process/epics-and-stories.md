# Epic and story workflow

## Epic lifecycle
1. Define problem, research questions, goals, scope, and success metrics.
2. Break work into stories with acceptance criteria and deliverables.
3. Identify critical path and dependencies.
4. Execute stories with reproducibility and evidence discipline.
5. Close epic only when required artifacts and quality gates are complete.

## Story lifecycle
1. Story states user value and acceptance criteria.
2. Story lists outputs, dependencies, risks, and references.
3. Story includes reproducibility notes for analytical or experimental outputs.
4. Medium/high complexity stories (`L2/L3`) run planning loop from `docs/process/agent-protocols.md`.

## Research quality checks
- All research stories cite primary sources where available.
- Distinguish sourced claims from inference.
- Note limitations and unresolved contradictions.
